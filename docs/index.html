quickstart.py
# Encrypt → consent check → audit log. Three lines.
from neuroguard import NeuralDataCipher, ConsentManager, AuditLogger
from neuroguard.consent import ConsentScope

# Encrypt on-device before anything else touches it
cipher = NeuralDataCipher(secret="user-passphrase")
encrypted = cipher.encrypt(neural_payload)

# Enforce consent before processing
consent = ConsentManager()
consent.grant(ConsentScope.PROCESSING)
consent.require_consent(ConsentScope.PROCESSING)

# Tamper-evident audit log — every action recorded
AuditLogger().log("ENCRYPT", actor="app", outcome="success")

The problem

Neural and biometric data is too sensitive for that model. By the time it reaches your infrastructure, the damage is already possible.

Core SDK

Everything you need.
Nothing you don't.

Four components. Drop them in, configure once, and your app is privacy-compliant by architecture — not by policy.

🔐
NeuralDataCipher
Symmetric AES-128-CBC + HMAC encryption for neural and biometric payloads. Generate keys or derive them from a user passphrase (PBKDF2). Data is encrypted on the device before any framework, network, or server can see it. Nothing leaves in plaintext — ever.
Built & tested
✅
ConsentManager
Grant, revoke, and enforce consent by scope — processing, export, analytics, and more. Call require_consent() before any sensitive operation and it raises if consent isn't active. Zero chance of accidental processing.
Built & tested
🔗
ConsentLedger
Append-only, hash-chained JSONL log of every consent grant and revoke. Each event includes hash_prev and hash_current. Call verify_chain() to detect tampering. Export as JSON for auditors or DPAs.
Built & tested
📋
AuditLogger
Structured, tamper-evident log entries for every sensitive action — actor, resource, outcome, timestamp. Compliance-grade logging without the infrastructure overhead.
Built & tested
🌐
Local REST API
FastAPI server with health, vault, consent, and compliance endpoints. Run locally. Integrates with any stack. Compliance reports and evidence bundles via HTTP.
Built & tested
📦
Evidence Bundle
One endpoint. One ZIP. PDF report + JSON audit + consent ledger + hash chain proof. Everything a regulator or DPA would ask for, generated in seconds.

How it works

Four steps. Privacy by architecture.

Not a policy. Not a checkbox. Privacy enforced at the code level, before data moves anywhere.

1
🔐
Encrypt
NeuralDataCipher wraps the raw payload in AES-128-CBC + HMAC. Key derived on-device. Nothing leaves plaintext.
2
✅
Check consent
ConsentManager enforces scope-level consent. require_consent() blocks the operation if consent isn't active.
3
📋
Log the action
AuditLogger records every sensitive operation with actor, resource, and outcome. Hash-chained. Tamper-evident.
4
📦
Report
Export compliance evidence as PDF, JSON, or a full cryptographically-verifiable ZIP bundle on demand.
